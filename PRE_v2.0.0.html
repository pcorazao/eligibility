    Name Type Required Restrictions Description
    rateSourceProcessingOrder string false none An Optional value that selects the data sources used in patien estimation. Default selection is Standard which runs through all sources.
    serviceFlag string true none 271 - Clients run eligibility and supply 271 with the PRE request. Manual- Clients run eligibility and supply benefits in the request. Selfpay- Clients identify the patient as a self pay

    benefit

    {
  "annualDeductible": 1000,
  "annualOutOfPocketMaximum": "2500",
  "annualOutOfPocketMaximumRemaining": "2000",
  "copay": "20",
  "coinsurancePercentage": 10,
  "deductibleRemaining": 100

    1. The benefits section is meant to be used when the user wants to supply benefits by manually entering them with service flag (customerPreference/serviceFlag) set to "Manual".The benefits in this section will be used only when the service flag is set to "Manual"
    2. If service flag is 271 , the client needs to supply a 271 file with the JSON request.
    3. The benefits are not required to be supplied and will be ignored in the estimation when the service flag is Self Pay.

    When requesting systems wants PRE to use the manually supplied benefits (FLAG set to MANUAL), follow the below steps: 1. If members coinsurance is 20%, send 20 and not 80. 2. Supply at least one of the two OOP Max or OOP Remaining, preferably both but atleast OOP Remaining. 3. Both annual deductible and deductible remaining are required fields

    copay string false none Not mandatory but Recommended when service flag is "Manual"
    coinsurancePercentage number false none Not mandatory but Recommended when service flag is "Manual"
    deductibleRemaining number true none Mandatory when service flag is "Manual"

    Properties

    Name Type Required Restrictions Description
    tradingPartnerServiceId string true none Supply Change Healthcare's ERA payer Ids. This list is available on Change Healthcare's website. https://access.emdeon.com/PayerLists/
    payerSequence string false none Required when the number of payers supplied is >1 i.e. patient is known to be enrolled with more than 1 insurer. In case of a single payer, default will be assumed to be primary
    planType string true none Claim Filing Indicator
    serviceTypeCodes [string] false none Multiple service type codes can be passed in the array when serviceFlag is 271 and client has supplied a 271 with the input request. The service type codes passed in this field are used to extract benefits from the 271. If service type codes are not supplied, the application uses its in built logic to scan and decide the service type codes to be used. The code/s supplied here will be used only when the service flag is "271" and a 271 file is attached with the request.

    Properties

    Name Type Required Restrictions Description
    firstName string false none A provider/physician's first name.
    lastName string false none A provider/physician's last name
    middleName string false none A provider/physician's middle name
    organizationName string false none Name of provider's practice/hospital/facility.
    placeOfService string true none A 2 digit code used when billing a claim. 1. In professional setting it is identified as a Place of Service code as published by CMS. Top 5 often used are 11- Office, 20-Urgent Care Facility, 22-Outpatient Hospital,32:Nursing Facility,49-Independent Clinic, 81: Independent Laboratory. Refer the complete list- http://www.cms.gov/Medicare/Medicare-Fee-for-Service-Payment/PhysicianFeeSched/Downloads/Website-POS-database.pdf 2. In Hospital setting, it is the 2 digit facility code
    npi string true none Provider's Billing NPI or Rendering NPI
    taxonomyCode string false none If not provided, the application product will retrieve it by looking up the codes published by NPPES and use the Primary taxonomy code of the provider. Refer CMS link for valid list of taxonomy codes to be supplied https://www.cms.gov/Medicare/Provider-Enrollment-and-Certification/MedicareProviderSupEnroll/Taxonomy.html
    lineOfBusiness string true none One of the two values Professional or Facility depending on the provider type. If a provider bills claim in CMS 1500, then use Professional. If billing on UB 04, use Facility
    taxId string true none Provider billing tax id
    postalCode string true none 5 or 9 digit zip (9 digit in the format XXXXX-YYYY) of the zip code that provider uses to bill the claims
